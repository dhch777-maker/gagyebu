월간 매출 파일들을 통합하고 대시보드와 보고서를 자동 생성하세요.

사용자 입력: $ARGUMENTS

## 수행 절차

### 1. 입력 폴더 확인
- `$ARGUMENTS`가 있으면 그 경로를 입력 폴더로 사용합니다.
- `$ARGUMENTS`가 없으면 사용자에게 월간 매출 xlsx 파일들이 있는 폴더 경로를 물어보세요.
- 폴더 안에 xlsx 파일이 몇 개인지 확인하고 파일 목록을 보여주세요.
- 12개가 아니면 경고하되 진행 여부를 확인하세요.

### 2. 통합 스크립트 실행
먼저 아래 명령어로 이 프로젝트의 루트 경로를 확인하세요.

```bash
git rev-parse --show-toplevel
```

확인된 프로젝트 루트를 `PROJECT_ROOT`로 사용해 아래 명령어를 실행하세요. `[입력폴더]`는 확인된 경로로 대체합니다.

```bash
cd "$(git rev-parse --show-toplevel)" && node 03_매출통합/generate_sales_report.js "[입력폴더]"
```

결과물은 입력 폴더의 상위 폴더에 자동 저장됩니다.
결과물 저장 위치를 별도로 지정하고 싶으면 두 번째 인자로 전달하세요.

```bash
cd "$(git rev-parse --show-toplevel)" && node 03_매출통합/generate_sales_report.js "[입력폴더]" "[출력폴더]"
```

### 3. 생성 결과 확인
스크립트가 완료되면 다음 3개 파일이 생성됩니다:
- `{연도}_연간매출_통합.xlsx` — 12개 파일 통합 (헤더 표준화, 서식 적용)
- `{연도}_연간매출_대시보드.xlsx` — 6개 분석 시트 + 차트 내장
- `{연도}_연간매출_보고서.pptx` — 11슬라이드 보고용 PPT

생성된 파일들이 실제로 존재하는지 확인하세요.

### 4. 자동 커밋
작업이 성공적으로 완료되면 생성된 파일들을 git add하고 커밋하세요.
커밋 메시지는 다음 형식을 사용합니다:

```
feat: {연도} 연간 매출 보고서 자동 생성

- 입력: {파일 수}개 월간 매출 파일 ({연도}.01 ~ {연도}.12)
- 통합 엑셀: 헤더 표준화 및 서식 적용
- 대시보드: 차트 6종 및 6개 분석 시트
- PPT: 11슬라이드 보고서 (인사이트 포함)
```

### 주의사항
- 헤더명이 다른 파일(한글/영문/다른 표현)도 자동으로 매핑됩니다
- 연도가 다른 파일이 섞여 있으면 경고가 표시됩니다
- 노드 패키지(xlsx, exceljs, chartjs-node-canvas, pptxgenjs)는 이미 설치되어 있습니다
- 스크립트 오류 시 에러 메시지를 확인하고 원인을 파악해서 해결해주세요
